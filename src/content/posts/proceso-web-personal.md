Esta primera entrada deja documentado el objetivo principal del proyecto: tener una web propia que sea rápida, escalable y suficientemente simple como para mantenerla sin depender de una base de datos ni de un constructor pesado.

## Qué quiero resolver

- Mostrar proyectos de distintas disciplinas sin mezclarlos todos en una sola vista.
- Escribir artículos tipo blog con imágenes, videos y archivos descargables.
- Publicar demos o utilidades web que puedan crecer con el tiempo.

## Qué cambia en mi flujo

Antes el contenido dependía mucho de herramientas con muchas capas encima. Ahora el objetivo es separar mejor:

- El **contenido** que escribo y subo.
- El **diseño** que organiza ese contenido.
- La **publicación** que convierte todo a un sitio estático listo para hosting compartido.

Esa separación es justamente la base de este proyecto.
